# COLD MEMORY / 冷记忆 数值系统开发需求

## 背景

当前原型仅支持以下状态：

- 航程
- 能量
- 当前章节
- 已解锁文本行数
- EVA 开关
- 自动巡航

这足以验证最基础的“巡航 - 解码 - 清障”循环，但无法承载新的 30 分钟数值设计。

因此需要新增一批系统能力，供开发实现。

## 必须新增的功能

### 1. 点数资源系统

目标：支持“记忆点”作为全局货币。

开发需求：

- 在全局状态中新增记忆点字段
- 支持记忆点增加、消费、保底发放、来源记录
- UI 中需要常驻显示当前记忆点、最近获得来源、当前自动产出速率
- recentEvents 需要支持记录“获得了多少点，来自什么行为”

验收标准：

- 普通剧情、战斗、探索、设备自动产出都能正确加点
- 任何消费都会即时反馈并更新 UI

### 2. 关键证词系统

目标：支持“战斗解锁关键剧情”的专属凭证。

开发需求：

- 在全局状态中新增关键证词库存
- 关键战斗胜利后固定发放 1 枚关键证词
- 关键剧情节点支持两种解锁条件：
  - 消耗 1 枚关键证词
  - 消耗指定数量记忆点

验收标准：

- 玩家能清楚看到关键剧情当前可否用证词解锁
- 玩家能选择“用证词”或“用点数”两种方式推进

### 3. 剧情节点类型化

目标：将当前“逐行文本”升级为“普通剧情节点 / 关键剧情节点 / 功能解锁节点”。

开发需求：

- 重构剧情数据结构，不再只用 content 字符串数组
- 每个剧情节点至少包含：
  - id
  - type
  - text
  - rewardPoints
  - unlockCondition
  - optionalFeatureUnlock
- 支持普通剧情节点发点数
- 支持关键剧情节点被单独锁定

验收标准：

- 不同类型剧情节点可独立显示、独立解锁、独立奖励
- UI 中能够明显区分“普通剧情”和“关键剧情”

### 4. 战斗事件类型化

目标：将现有 EVA 从单一用途升级为“日常作业”和“关键战斗事件”两类。

开发需求：

- 在战斗入口前增加事件配置
- 区分：
  - 日常战斗：主要给点数或能量
  - 关键战斗：给关键证词并推进章节
- 支持不同战斗事件使用不同目标击破数、不同奖励、不同文案

验收标准：

- 同一个 EVA 面板能承载至少两种奖励逻辑
- 章节推进不再只绑定“航道阻塞后打一场固定战斗”

### 5. 日常探索系统

目标：新增低压补充循环，让玩家在不打战斗时也有稳定资源来源。

开发需求：

- 新增探索入口与独立界面
- 支持探索层级 T1 到 T5
- 每次探索具备：
  - 持续时间
  - 固定奖励
  - 波动奖励
  - 解锁章节要求
- 支持探索完成后的统一结算

验收标准：

- 玩家能在剧情与战斗外稳定获得记忆点
- 探索结果可被升级系统放大

### 6. 设备升级系统

目标：让记忆点能转化为自动点数能力。

开发需求：

- 新增设备商店或升级面板
- 至少支持 4 条设备线：
  - 档案继电器
  - 勘探无人机
  - 回收阵列
  - 解码缓存器
- 每条线至少 3 级
- 每级需配置：
  - 成本
  - 产出效果
  - 解锁章节

验收标准：

- 升级后自动点数收益立刻可见
- 状态面板能显示当前总自动点数每秒收益

### 7. 自动点数结算逻辑

目标：让自动获取点数真正成为中后期节奏核心。

开发需求：

- 支持常驻每秒收益
- 支持只在自动巡航中生效的每秒收益
- 支持后台解析类每秒收益
- 所有自动收益应以统一 tick 或统一结算函数实现，避免多个定时器彼此打架

验收标准：

- 自动点数不会因为 UI 重绘而丢失或重复结算
- 自动收益来源可拆分展示

### 8. 数值配置化

目标：避免后续平衡仍然硬编码在控制器中。

开发需求：

- 将下列数值改为配置表驱动：
  - 普通剧情奖励
  - 关键剧情直购成本
  - 战斗奖励
  - 探索奖励
  - 设备升级成本与收益
  - 章节目标时长参考值
- 建议放入 src/data 下独立配置文件

验收标准：

- 策划调整数值时不需要修改控制流程代码

### 9. 30 分钟测试与埋点支持

目标：验证本次数值方案是否真的能打到 30 分钟通关。

开发需求：

- 记录以下统计：
  - 首次通关总时长
  - 各章节耗时
  - 普通剧情平均解锁间隔
  - 关键战斗胜率
  - 探索平均使用次数
  - 各设备购买时点
  - 关键剧情被“证词解锁”还是“点数直购”

验收标准：

- 单局结束后，开发环境可查看完整节奏日志

## 建议的数据结构调整

建议至少新增以下数据层：

- story-nodes.js：剧情节点配置
- battles.js：关键战斗与日常战斗配置
- exploration.js：探索层级、奖励与持续时间配置
- upgrades.js：设备升级配置
- economy.js：奖励倍率、保底规则、章节目标时长配置

## 当前版本不满足的点

与当前原型相比，以下能力尚不存在：

1. 没有点数货币
2. 没有剧情奖励机制
3. 没有关键剧情与普通剧情的类型区分
4. 没有日常探索
5. 没有升级商店
6. 没有自动点数系统
7. 没有关键证词
8. 没有配置化平衡表

所以本次内容先以策划与需求文档形式落地，后续需由开发先补系统，再进行第二轮数值实装。
