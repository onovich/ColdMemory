# COLD MEMORY / 冷记忆 游戏设计梳理

## 定位

- 类型：太空惊悚题材的放置点击游戏
- 气质：复古工业终端、孤独航行、叙事驱动、低频高压惊悚感
- 当前目标：先用 Web 原型验证主循环与叙事节奏，再迁移到 Unity

## 核心体验

- 玩家扮演在异常飞船中醒来的克隆体
- 通过终端逐步恢复记忆并判断自身处境
- 在自动推进与手动干预之间切换节奏
- 用有限的互动承载强叙事和心理压力

## 当前主循环

1. 启动自动巡航，消耗能源并推进航程
2. 手动点击解析记忆片段，逐步揭示章节文本
3. 到达章节阻塞点后，进入 EVA 清障小游戏
4. 完成清障后推进下一章节，继续揭露真相

## 系统拆分

### 数据层

- 章节脚本
- 数值常量
- 章节触发阈值

### 逻辑层

- 全局状态存储
- 自动巡航 tick
- 记忆解析进度
- 章节阻塞判断
- EVA 完成后的推进结算

### 表现层

- Boot 启动画面
- 舰桥终端 HUD
- 章节文本显示
- EVA Canvas 清障界面
- 响应式静态页面样式

## 后续适合扩展的设计方向

- 新资源：理智、船体完整度、通信噪声、冷冻液库存
- 新玩法：舱段调查、终端日志破译、多结局分支
- 新叙事：母亲 AI 与克隆体身份冲突、现实层与模拟层错位
- 长线留存：离线收益、研究升级、章节外随机事件

## Unity 移植建议

- 数据保持 JSON 或 ScriptableObject 配置化
- 逻辑状态机独立于 UI 视图
- EVA 小游戏拆成独立场景或 Overlay 面板
- 文本叙事与动画表现解耦，避免内容更新需要改逻辑代码
