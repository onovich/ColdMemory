# 后续计划

## Phase 1: Web 原型打磨

1. 增加存档与读取
2. 落地记忆点、关键证词、设备升级与自动收益系统
3. 落地日常探索与关键剧情分型
4. 为 EVA 加入关键战斗事件与差异化奖励

## Phase 2: 内容与系统验证

1. 把剧情节点、探索、战斗、升级全部抽成配置表
2. 按 30 分钟通关目标做一轮首轮平衡测试
3. 梳理结局分支、关键剧情直购与战斗分支
4. 补充埋点和开发调试面板，记录单局节奏数据

## Phase 3: Unity 移植准备

1. 定义状态机与数据表结构
2. 确定 UI 分层与场景切分
3. 验证输入系统、存档系统、成长系统和动画流程
4. 保留 Web 原型作为叙事、数值和节奏对照基线
