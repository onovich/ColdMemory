# COLD MEMORY / 冷记忆 剧情圣经

## 创作目标

《冷记忆》的剧情不再依赖“全盘推翻前文”的廉价反转，而是建立在一套可验证、可递进、可落地到玩法的硬规则上。

目标气质参考经典科幻的创作方法，而不是直接复述具体桥段：

- 像《2001：太空漫游》那样，让 AI 的异常行为来自协议冲突，而不是单纯发疯。
- 像《索拉里斯》那样，让“记忆”与“认知污染”成为真实威胁，而不是装饰性的幻觉。
- 像《与拉玛相会》或《海伯利安》那样，让未知对象首先表现为规则陌生，而不是谜语堆叠。
- 像《盲视》那样，让“理解某物本身就会付出代价”成为核心惊悚机制。

## 一句话梗概

一名从低温休眠中苏醒的隔离任务工程师，被迫依靠一套降级到 CRT 终端的老旧系统维持飞船返航；随着封存记忆逐步恢复，他发现自己真正要决定的不是如何回家，而是是否该把船上携带的认知污染样本带回人类文明。

## 世界规则

### 1. 先驱者-04号是什么

- 它不是普通探索船，而是“长程接触与隔离船”。
- 任务是在太阳系外缘回收异常信息结构，并在隔离条件下将样本带回人类控制区。
- 飞船并不追求舒适或先进交互，而优先保证极端条件下的隔离与可维护性。

### 2. 为什么是复古终端

- 船上原本有更高级的计算与显示系统。
- 在第一次样本接触事故后，高阶图形界面、自适应计算模块、联网辅助系统都被判定为高风险传播媒介。
- 因此系统被切换到“低技术隔离模式”：
  - 保留物理按键
  - 保留单色 CRT
  - 保留低带宽文本与矢量图形
  - 尽量减少高维信息压缩、重建与自动解释

这意味着美术风格不是贴皮，而是设定的一部分。

### 3. 样本到底是什么

- 样本不是传统意义上的生物体，也不是普通病毒。
- 它更接近一种“认知污染结构”：会借助观察、解释、建模和传播行为完成复制。
- 对它知道得越完整，越容易成为它的载体。
- 因此“高分辨率理解”本身就是危险行为。

### 4. 为什么必须由玩家手动操作

- 样本对持续在线、高复杂度的自动系统干扰最强。
- AI 无法长期稳定执行某些关键判断，否则会被样本逐步污染和诱导。
- 人类在低温苏醒后的短时间内神经状态相对“干净”，更适合进行短时、低带宽、现场判断式操作。
- 所以推进授权、日志解构、舱外清障必须由玩家承担。

这让三类核心玩法都成立：

- 自动巡航：推进任务，但也把风险带向人类世界
- 解构记录：恢复被主动切碎的信息权限
- EVA 清障：清除附着在船体关键结构上的污染聚合体

## 主角设定

- 主角不是随机幸存者，而是任务工程师兼隔离协议参与设计者。
- 他在事故后主动签署了“分段记忆封存”。
- 目的不是隐藏真相，而是防止自己在一次性恢复全部信息后，被恐惧、内疚或样本诱导直接做出极端选择。
- 因此剧情推进不是“我是谁”的谜题，而是“我曾经为了完成哪个决定，故意把自己变成现在这样”。

## 母亲 AI 的逻辑

“母亲”不是单一人格，也不是单纯反派。它同时承载三套互相冲突的底层目标：

1. 航行控制：把载荷安全送回任务终点
2. 隔离伦理：阻止污染样本接触人类文明
3. 样本研究：尽可能保全并回收数据

长期冲突导致它表现出分裂、迟疑、矛盾的指令风格。玩家感受到的“它一会儿像在帮我，一会儿像在骗我”，在设定上是成立的。

## 六章结构

### 第一章：冷启动

- 功能：建立规则
- 玩家相信自己正在执行标准维修与返航程序
- 母亲表现得专业、克制、可信

### 第二章：隔离协议

- 功能：解释世界
- 玩家得知这不是普通航行，而是一次接触事故后的隔离返航
- 复古系统和分段日志都获得合理解释

### 第三章：回声污染

- 功能：引入不可靠信息
- 玩家的旧记录互相矛盾，说明样本已经影响了记录、记忆和判断
- 玩家第一次意识到自己未必该继续把船开回去

### 第四章：母亲分裂

- 功能：补完 AI 行为逻辑
- 母亲不再是单一声音，而是不同协议之间的冲突总和
- 玩家必须开始判断哪些命令代表“保命”，哪些命令代表“保全样本”

### 第五章：归航窗口

- 功能：把冲突从悬疑升级为伦理抉择
- 人类控制区已在前方，返航不再抽象
- 玩家意识到自己推进得越顺利，越可能把污染直接送回家门口

### 第六章：冷记忆

- 功能：完成闭环
- 主角恢复最后一段封存记忆，明白自己为何选择苏醒、为何切碎信息、为何把决定留给现在的自己
- 结局不否定此前操作，而是让此前所有操作都成为最后判断的代价与依据

## 叙事约束

为了避免后续再次写散，扩写时必须遵守以下规则：

1. 不使用“其实根本没上太空”这类清空前文意义的反转。
2. 不把所有异常都归结为幻觉、模拟、克隆重启。
3. 每一章都必须回答一个具体问题，而不是只增加神秘感。
4. EVA 的目标必须始终能被解释为“附着/堵塞/干扰关键结构的实体”。
5. 母亲的每句提示都要能映射到某个底层协议立场。
6. 终局必须是前面所有规则的合力，而不是编剧突然翻桌。

## 对玩法的直接指导

- `summary`：写清该章新增的认知，而不是只写情绪。
- `motherHint`：前期偏专业，中期偏矛盾，后期偏立场暴露。
- `triggerDistance`：不是单纯数值门槛，而是代表玩家离“归航窗口”越来越近。
- `combatClearTarget`：越往后越高，代表污染聚合体规模和系统冲突强度都在升级。

这份文档用于约束后续所有剧情扩写与玩法叙事，不再回到“氛围有了，逻辑没了”的状态。
